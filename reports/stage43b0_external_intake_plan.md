# Stage43-B0 External Validation Intake Plan

Preparation only. No model training, evaluation, external probe evaluation,
Kaggle command, smoke test, or local model execution was performed to
produce this plan.

## 1. Decision

**Decision:** `STAGE43B0_EXTERNAL_INTAKE_PLAN_PREPARED`

## 2. Background

Stage43-A ([`reports/stage43a_external_validation_manifest.json`](stage43a_external_validation_manifest.json),
[`reports/stage43a_external_validation_manifest.md`](stage43a_external_validation_manifest.md))
found no clearly adaptable naturalistic external validation source in the
current repository:

- decision: `STAGE43A_EXTERNAL_VALIDATION_MANIFEST_AMBIGUOUS`
- 34 candidate files scanned
- 0 clearly adaptable
- 10 ambiguous
- 24 not_adaptable
- no VitaminC/FEVER/RTE/MNLI/SNLI-style file exists in the repo
- all schema-plausible candidates are synthetic/controlled probes

**Conclusion:** Stage43-B cannot honestly proceed until a naturalistic
external claim/evidence/label source is added to the repository or
supplied by the user.

## 3. Preferred Source Order

1. EpistemicBERT VitaminC confident-error or labeled claim-evidence set, if available.
2. VitaminC/FEVER-style claim-evidence data.
3. RTE/MNLI/SNLI subset.
4. A small manually curated external validation set, only if labels are
   independently assigned and not generated by ContraMamba.

## 4. Converter Scope

[`scripts/convert_stage43_external_validation.py`](../scripts/convert_stage43_external_validation.py)
converts a user-supplied naturalistic claim/evidence/label dataset into the
ContraMamba Stage43 external evaluation schema documented in
[`docs/stage43_external_validation_schema.md`](../docs/stage43_external_validation_schema.md).

Supported input formats: `.jsonl`, `.json`, `.csv`, `.tsv`.

The converter only prepares evaluation files. Converted output must not be
used for training, calibration, threshold selection, checkpoint selection,
or loss design.

## 5. Stage43-B Evaluation Plan (Future, Not Executed Here)

Once a naturalistic external source is available and converted, Stage43-B
should evaluate **Stage39-C safe_structured_v2** on the converted file,
without modifying model or training code. Stage43-B must not:

- modify model code
- modify training scripts
- modify evaluator scripts
- modify Stage33/36/37/39/40 logic
- modify existing Stage39/40/41/42/43-A reports
- modify existing data/probe files

## 6. Guardrails

Stage43-B0 (this preparation layer) and any future Stage43-B run must not:

- download external datasets
- create fake examples
- infer labels from model outputs
- use synthetic Stage34/35 probes as naturalistic external validation
- alter source files
- run training
- run evaluation
- add generated converted data unless an input file is actually provided

## 7. Leakage Policy

This intake layer (schema + converter) is preparation-only. No conversion
was run and no converted data files were generated as part of Stage43-B0,
because no external input file was supplied. If and when a naturalistic
external source is added or supplied, converted output must remain
external-evaluation-only and must never be used for training, calibration,
threshold selection, checkpoint selection, or loss design.

## 8. Recommendation

Await a genuinely external/naturalistic claim-evidence source (per the
preferred source order above) before running
`scripts/convert_stage43_external_validation.py` or any Stage43-B
evaluation. No model, training, or evaluator code was modified to produce
this plan.
